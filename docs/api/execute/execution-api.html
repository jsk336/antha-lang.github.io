Execution API page

The execution context structure is globally accessible and allows all services a channel by which they can talk to other services.

The following service interfaces are required:

  • SampleTracker : CRUD tool for talking to database of samples in system
  • StockManager : CRUD tool for talking to database of stocks
  • Logger : Simple logging daemon
  • Scheduler : Handles requests for movement, liquid handling and other device interaction
  • GarbageCollector : Holds a reaper queue

Additionally, the execution context must hold the following channels, which should only be accessible directly by the scheduler

  • LiquidHandler : Logical interface for requesting any kind of liquid handling operation
  • Mover : Logical interface for requesting movement of entities
  • Platereader : Logical interface to spectrophotometry
  • Store : Logical interface to storage devices
